# src/ — 소스코드 루트

**R11 (소스 경계)**: 모든 소스코드 작업은 `./src/` 안에서만 수행한다. `src/` 밖에는 코드 파일을
만들지 않는다 (`data/`는 데이터 산출물, `docs/`는 문서 전용).

모든 직원(에이전트)은 폴더에서 작업을 시작하기 전에 **해당 폴더의 `README.md`를 먼저 읽고**
그 작성규칙을 따른다. 규칙이 티켓 명세와 충돌하면 작업을 멈추고 부장에게 보고한다(R6).

## 폴더 맵 (소유 팀 = 쓰기 권한, 조직도 §3)

| 폴더 | 소유 팀 | 내용 |
|---|---|---|
| `src/backend/` | 코드설계팀 | FastAPI 백엔드 — 클린 아키텍처 (architecture.md §②) |
| `src/frontend/` | 코드설계팀 | 정적 프론트 (GitHub Pages, 빌드 스텝 없음) |
| `src/tests/` | 코드설계팀 | 백엔드 단위·계약 테스트 |
| `src/scripts/` | 데이터팀 | 데이터 가공·운영 툴 (표준 라이브러리만) |

소유 팀이 아닌 팀은 해당 폴더를 **읽기 전용**으로만 사용한다. 위반은 검수(R6)에서 반려된다.
