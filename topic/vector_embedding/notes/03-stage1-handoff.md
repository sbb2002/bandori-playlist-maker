# Stage 1 인수인계 (2026-07-17, 서브 로컬 → 메인 로컬)

Stage 2(키워드 기반 쿼리 재설계·검색 재평가)는 메인 로컬에서 진행 예정. 이 문서는 그
세션이 Stage 1 결과를 빠르게 파악하기 위한 요약이다. 설계 배경·전체 맥락은
`DESIGN.md` §1b, 원자료는 `out/song_profiles.csv`/`out/profile_qc_sheet.csv` 참조 —
여기서는 **거기 없는 종합·해석**만 남긴다.

## 무엇을 했나

`02b_profile_songs.py`로 14곡(pool)을 각각 (1) 한 문장 감성 요약(desc) (2) 지배적
키워드(keyword_main) (3) 2차 키워드(keyword_sub)로 프로파일링하고, 연구자가 "desc/키워드가
실제 곡을 정확히 대표하는가"를 1~5점으로 QC 채점 완료(RQ1 arm 비교와는 무관한 별도 검증
단계 — `DESIGN.md` §1b 참조).

**중요한 제약**: 이 실행에 쓴 서브 로컬 기기엔 가사 원문(`work/transcripts/`)이 없어서(다른
기기 산출물, 저작권상 미커밋), 원칙적 입력(`PROFILE_PROMPT`, 원문 가사) 대신 이미 커밋된
`texts_summary.csv`(LLM 무드 요약)를 입력으로 쓰는 대체 프롬프트(`PROFILE_PROMPT_FROM_SUMMARY`)
로 진행했다. `song_profiles.csv`/`profile_qc_sheet.csv`의 `source` 컬럼이 14곡 전부
`summary-fallback`이다 — "요약의 요약"이라 원문 직접 분석보다 충실도가 낮을 수 있다.

## 결과

- 평균 4.0/5, P@5(4점 이상 비율) 9/14 = 64.3% — v1 파일럿(고정 카테고리, 평균 2.9 내외)보다
  높음. 단, 채점 질문 자체가 다르므로(v1="사용자 요청과 맞는가" vs Stage 1="곡을 정확히
  대표하는가") 절대 비교는 주의.
- 최저점 1곡(ave_mujica KiLLKiSS, 1점), 2점 1곡(mugendai_mutype どんがらがっしゃん) 제외하면
  대부분 3~5점.

## 코멘트에서 반복 발견된 패턴 (Stage 2 설계 시 참고)

1. **main/sub 키워드 순서 뒤바뀜**: 14곡 중 4곡(29%, p001/p008/p009/p014)에서 "메인과
   서브가 뒤바뀐 것 같다"는 코멘트. LLM이 실제 지배적 감정보다 부차적 감정을 MAIN으로
   뽑는 경향이 종종 있음. Stage 2에서 순서를 엄격히 신뢰하는 로직(예: MAIN만 쿼리 앵커로
   씀)은 위험할 수 있음 — MAIN/SUB를 순서 무시하고 "이 곡의 키워드 쌍"으로 묶어 쓰거나,
   프롬프트에 재검토 단계를 추가하는 걸 고려할 것.
2. **가사와 곡조가 어긋나는 곡은 점수 급락**: 최저점 두 곡 모두 코멘트가 "가사/키워드는
   맞는데 곡조(헤비메탈·밝은 멜로디 등)가 반대 인상"이라고 명시. 가사 전용 방법론의
   구조적 한계(오디오 미사용, DESIGN.md §8)가 실제로 이 두 건에서 점수를 깎았다 — Stage 2
   결과 해석 시 "저조한 곡은 라벨이 틀려서"가 아니라 "가사·사운드 불일치 곡일 수 있다"는
   가능성을 먼저 확인할 것.
3. LLM 출력에 간헐적으로 한자가 섞이는 버그 1건 발견·수정(`흥奋`→`흥분`, temperature=0이라
   재시도로 안 고쳐져 수동 정정) — Stage 2 프롬프트에도 유사 이슈 가능성 있으니 표본 검수
   습관화 권장.

## Stage 2로 넘어가기 전 확인 사항

- `song_profiles.csv`의 `keyword_main` 분포: 쓸쓸함 계열(직접/유사어)이 5곡으로 가장 많고
  나머지는 흩어져 있음 — 그러나 pool이 14곡뿐이라 클러스터라 부르기엔 표본이 작다. Stage 2
  쿼리 앵커를 이 분포에서 뽑을지, pool을 먼저 늘릴지는 미결정(메인 로컬에서 판단).
  `DESIGN.md` §1b "Stage 2 설계 원리" 절에 메커니즘(전역 최고-cosine 배정 규칙 재사용 등)은
  이미 문서화돼 있음 — 구체적 쿼리 문구·앵커만 미정.
- 원칙 입력(원문 가사)으로 재실행할지: 메인 로컬에 `work/transcripts/`가 있다면
  `PROFILE_PROMPT`(원문 기반)로 다시 돌려 `summary-fallback` 라벨을 정본으로 교체하는 걸
  권장. 없다면 이번 결과를 그대로 Stage 2 근거로 써도 QC 통과 수준(평균 4.0)이라 무리는
  없어 보임.
