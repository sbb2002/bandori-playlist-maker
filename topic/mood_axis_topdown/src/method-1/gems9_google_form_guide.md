# 구글폼 자동 생성 스크립트 사용설명서 (gems9_google_form.gs)

## 무엇을 만드는가
35곡짜리 GEMS-9 설문 구글폼을 한 번에 생성. 곡마다 페이지 하나(영상 문항 + 9항목
그리드 채점 + 인상 메모)로 구성됨. n≥20 다중 응답자 라운드용 — 응답은 자동으로
구글시트 하나에 실시간 수합됨(각자 CSV 내보내서 보낼 필요 없음).

## 실행 방법
1. [script.google.com](https://script.google.com) 접속 → 새 프로젝트.
2. `gems9_google_form.gs` 내용을 그대로 붙여넣기.
3. 상단 함수 선택 드롭다운에서 `buildForm` 선택 → 실행(▶) 클릭.
4. 처음 실행 시 권한 승인 창이 뜸 — 본인 계정이므로 승인.
5. 실행 로그(보기 → 로그, 또는 Ctrl+Enter)에서 두 링크 확인:
   - **편집 링크**: 폼 내용을 직접 손볼 때 사용.
   - **응답 링크**: 이 링크를 응답자들에게 공유.
6. 응답 결과는 폼 우측 상단 "응답" 탭 → 스프레드시트 아이콘으로 구글시트 연결.

## 영상 재생 방식 — 구간 지정 없이 전곡 재생 (2026-07-23 결정)
- `setVideoUrl`에 영상 ID만 단독으로 넣는다(`?start=`, `embed/...?start&end` 등은 실제
  테스트 결과 Forms가 쿼리스트링을 못 파싱하고 영상 ID에 그대로 이어붙여 영상 자체가
  깨짐 — 검은 화면, 재생 불가). 즉 Forms에서 구간을 기술적으로 강제할 방법이 없다.
- 대신 report/02(라운드2 confound 체크)에서 인트로 구간과 하이라이트 구간으로 바꿔
  재채점해도 GEMS-9 9항목 전부 유의한 차이가 없었다(전부 p>0.2) — 구간 선택이 점수에
  큰 영향을 주지 않는다는 근거가 있어, 구간 강제를 포기하고 전곡을 재생하기로 했다.
  응답자는 원하는 만큼만 듣고 채점해도 된다(폼 설명문에 안내).
- 커스텀 툴(Tool 2, `segment_survey_tool.html`)만큼의 정밀한 구간 통제는 없지만, 애초에
  구간 통제 자체의 실익이 크지 않다고 판단했고, 다중 응답자 운영·자동 집계는 그대로
  구글폼 쪽이 훨씬 쉽다.

## 곡 목록이 바뀌면 다시 만들어야 함
1. `out/gems9_pilot_candidates.csv`(곡 목록)가 바뀌면 `python build_gems9_survey_data.py`
   실행 — 이 스크립트 안의 `SONGS` 데이터가 자동 갱신됨(직접 편집 불필요).
2. script.google.com에서 갱신된 코드로 다시 붙여넣고 `buildForm` 재실행.

**주의**: `buildForm`은 매번 **새 폼**을 만든다(기존 폼을 고치지 않음). 이미 응답자에게
링크를 보낸 뒤라면 재실행하지 말 것 — 재실행하면 별개의 새 폼·새 응답 시트가 또 생김.

## 응답자 이름 항목
폼 맨 앞에 "응답자 이름 또는 별명"을 필수로 받음 — 나중에 라벨러 간 일치도(Cohen's
kappa 등, `framework.md` §2b)를 계산하려면 응답자 구분이 필요하기 때문.
