# 음악 감정 평가(GEMS) 실험 및 웹 설문 설계 표준 가이드

---

## 1. 개요 (Overview)
* **목적**: [Geneva Emotional Music Scale (GEMS)](https://en.wikipedia.org/wiki/Geneva_Emotional_Music_Scale)을 활용하여 특정 음악이 유발하는 감정 상태를 통계적으로 신뢰성 있게 측정하고 분석함.
* **평가 방식**: 음악 감상 후 자가 보고(Self-report) 방식의 설문 조사 (5점 리커트 척도 사용).

---

## 2. 피설문자(표본) 구성 표준

| 구분 | 피설문자 수 | 유효성 및 권장 용도 |
| :--- | :---: | :--- |
| **단일 대상 (1명)** | 1명 | **통계적 유효성 없음** (단일 사례 연구/질적 인터뷰 용도로만 활용) |
| **최소 통계 기준** | **10명 ~ 20명** | 특정 음악 자극에 대한 감정 점수가 통계적으로 안정화되는 **최소 기준** |
| **소규모/Pilot 실험** | **30명 이상** | 중심극한정리에 의해 정규분포 가정이 가능해지는 **권장 최소 기준** |
| **학술/마케팅 연구** | **100명 이상** | 집단 간 비교 분석 및 높은 신뢰도를 보장하는 정밀 연구 |

---

## 3. 음원 자극 및 구간 설정 (Stimulus Control)

### 3.1. 기본 원칙
1. **동일 음원 조건**: 모든 피설문자는 가창자, 편곡, 음질이 완벽히 동일한 음원을 감상해야 함.
2. **동일 구간 통제**: 위치를 무작위(Random)로 제시하거나 피설문자가 임의의 구간을 듣게 해서는 안 됨.

### 3.2. 대표 구간(Representative Excerpt) 선정
* **선정 주체**: 연구자(실험 설계자)가 직접 곡의 음악적 특징을 고려하여 지정함.
* **장르별 선정 기준**:
  * **대중가요 / 팝 / EDM**: 곡의 핵심 감정과 메인 멜로디가 축약된 **코러스(Chorus/후렴구/싸비)**.
  * **클래식 / 연주곡 / OST**: 주요 테마 선율이 집약된 **도입부 테마(Intro Theme)** 또는 **클라이맥스**.
* **재생 시간 (Duration)**: **30초 내외** (최소 15초 ~ 최대 45초 권장).

---

## 4. 웹 설문(YouTube IFrame API) 시스템 구현 표준

### 4.1. 기술적 제어 방안
* **구간 자동 제어**: API 매개변수(`startSeconds`, `endSeconds`) 또는 타이머 함수(`setTimeout`)를 활용해 지정된 구간 재생 후 자동으로 재생을 일시정지/정지하도록 구현.
* **UI/UX 통제**:
  * 피설문자가 임의로 탐색 바(Progress bar)를 조작하지 못하도록 Player UI 제어.
  * 중복 재생으로 인한 오차 방지를 위해 재생 시작 시 버튼 비활성화(`disabled`) 처리.
* **브라우저 정책 대응**: 사용자의 직접적인 클릭 이벤트(예: '음악 듣기' 버튼 클릭)를 통해서만 재생되도록 설계 (Autoplay 제한 방지).

---

## 5. 실험 절차 (Protocol Summary)

```text
[1. 사전 검증 (Pilot Test)]
   - 연구자가 선정한 대표 구간(30초)에 대해 소수 인원으로 사전 확인

[2. 실험 진행 (Web Task)]
   - 피설문자가 '음악 듣기' 버튼 클릭
   - YouTube IFrame을 통해 지정된 30초 구간 자동 재생 및 정지

[3. 감정 평가 (GEMS Survey)]
   - 감상 직후 GEMS 항목(GEMS-9, 25, 45 중 선택)에 5점 척도 응답

[4. 데이터 집계 및 분석]
   - 평균 점수 산출을 통해 숭고함(Sublimity), 활력(Vitality), 불안(Unease) 차원 분석
