# 배경

에너지 관련 제어 연구를 했었는데, 현재 배포판에 비해 특출나게 뛰어난 제어법을 찾지 못했다.
그래서 모델 요청 1회에 모든 것을 처리하기에는 역부족이라고 생각했다. 그래서 각 파라미터마다 순서대로 llm 요청을 보내 json을 완성하는 식으로 구성해보고자 한다.


# 구현방법
- 현재 아키텍쳐 개요 : 사용자 프롬프트 -> (LLM 제어 시) 파라미터를 한번에 제어 -> 작성된 파라미터로 플리제작
- 개량 아키텍쳐 개요 : 
    사용자 프롬프트 
    -> (1차제어) 프롬프트를 보고 이번 플리는 몇 분으로 설정하는게 좋을까? 정수로만 답해.
    -> (2차제어) 프롬프트를 보고 전체를 몇개 구간으로 나누고, 각 구간의 길이, 각 구간은 어떤 느낌(감정 키워드로 답변요청)은? 정수, 정수, 텍스트로 답해.
    -> (3차제어) 앞서 제시한 조건으로 각 구간의 느낌을 보고 에너지 값을 얼마로 하는게 좋을까? 0~1 사이의 소수점 2자리까지 float으로 답해.
    -> json 조립 후 이걸로 플리제작
