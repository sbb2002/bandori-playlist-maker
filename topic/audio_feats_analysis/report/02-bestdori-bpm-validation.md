# BPM 단일 후보 선출 + bestdori 공식 BPM 정량 검증

`report/01-tempo-octave-ambiguity.md` §6.2에서 예고한 두 작업의 실행 결과다.
서브 로컬(오디오 데이터 없음)에서 기존 CSV 컬럼과 bestdori API만으로 수행했다.
코드: `src/method-2/` (01_select_bpm → 02_fetch_bestdori → 03_compare).

## 1. 방법

### 1.1 단일 후보 선출 파이프라인 (661곡 전체)
- 후보: `drum_tempo_bpm × 2ⁿ` (n=−1..1) 중 [85,220] 범위 내 값. ±2옥타브 후보가 범위에
  드는 곡은 0곡이라 실질 후보는 {÷2, base, ×2} 세 개다.
- 선출 규칙(τ=0.96, 형제 프로젝트와 대칭 통일):
  - 후보 1개 → 그대로 채택(`unique`, 324곡 — base가 범위 밖이라 접힌 74곡 포함)
  - {base, ×2} → `pulse_ratio ≥ τ`면 ×2 (`acf_up`, 327곡)
  - {base, ÷2} → `pulse_ratio_down ≥ τ`면 ÷2 (`acf_down`, 10곡)
- 오디오 재계산 없음 — `audio_feats.csv`의 기존 ACF 컬럼만 사용.

### 1.2 bestdori 공식 BPM 수집·매칭
- `all.5.json`(804곡)에서 bandId+정규화 제목(NFKC·casefold·공백 제거·〜/～ 통일)으로
  1차 매칭, `(Cover)` 접미사 제거 → 말미 괄호구 제거 → 전역 유일 제목 순으로 완화.
- 매칭 573/661곡(86.7%). 미매칭 88곡 = 게임 미수록 밴드(mugendai_mutype 등) 30곡 +
  제목 미발견 58곡(Ave Mujica·MyGO 신곡 등 게임 미구현 중심).
- 공식 BPM = 곡별 API의 bpm 세그먼트에서 구간 길이 가중 지배값. BPM 변화 있는 곡은
  65곡(11.3%)이며 부분집합 분리 검증 결과 전체와 차이 없음.

## 2. 결과

### 2.1 현행 ACF τ 규칙 (n=573)
| 지표 | 값 |
|---|---|
| Accuracy1 (옥타브 불허, ±4%) | **53.4%** |
| Accuracy2 (옥타브 허용, ±4%) | **96.9%** |
| strict 오차% mean±std | **−19.045 ± 30.013** |
| 옥타브 접은 오차% mean±std | **+1.115 ± 5.816** (\|오차\| 평균 1.563, 중앙값 +0.114) |

**해석**: 주파수 정밀도 자체는 우수하다 — 옥타브만 맞으면 공식 BPM과 평균 1.6% 이내로
일치한다(Accuracy2 96.9%). 오차의 대부분은 **옥타브 선택 실패**에서 나온다(573곡 중 260곡).

### 2.2 진단 — pulse_ratio는 옥타브 방향 판별력이 없다
정답 옥타브(공식 BPM 기준, ±4%) 분포: ×2가 338곡, base 215곡, ÷2가 2곡,
어느 옥타브로도 불일치(base 주파수 자체 오답) 18곡(3.1%).

- `acf_up` 모호곡 278곡(정답 판정 가능분) 중 **96%가 ×2 정답** — 드럼 스템 beat_track이
  체계적으로 절반 옥타브를 뽑는다.
- 정답이 base인 그룹의 `pulse_ratio`(mean 0.916)가 ×2 그룹(0.860)보다 **오히려 높다** —
  방향이 반대라 어떤 τ로도 못 가른다. τ 스윕(0.80~1.00) 전 구간에서 "항상 ×2"(95.7%)에
  미달함을 확인했다.
- `perceptual_pulse()`의 τ는 체감 pulse(세분화) 검출용으로 튜닝된 것이지 템포 옥타브
  판정용이 아니다 — 목적이 다른 지표를 전용한 것이 실패 원인으로 판단된다.

### 2.3 대안 — 최대 후보 규칙 (범위 내 후보 중 높은 옥타브 채택)
| 지표 | ACF τ 규칙 | **최대 후보 규칙** |
|---|---|---|
| Accuracy1 (±4%) | 53.4% | **92.5%** |
| strict 오차% mean±std | −19.045 ± 30.013 | **+2.830 ± 20.454** |

규칙별 Accuracy1: `acf_up` 94.7%, `unique` 90.5%(범위 밖 접힘 71곡도 91.5%), `acf_down` 87.5%.
잔여 오답 43곡: 2배 초과 19곡, 절반 16곡, base 주파수 자체 오답 8곡.

## 3. 결론

1. **방법론의 주파수 정밀도는 검증됨** — 옥타브 접은 오차 mean +1.1%, std 5.8%,
   중앙값 +0.1%. 드럼 스템 ACF 기반 템포 추정 자체는 신뢰할 수 있다.
2. **옥타브는 ACF 신호로 판정 불가** — report/01 §3.4(悪魔の子)에서 의심된 한계가
   전수 검증으로 확정됐다. τ 기반 대칭 규칙은 폐기한다.
3. **"[85,220] 내 최대 후보" 단순 규칙이 실용 최적** — Accuracy1 92.5%. 이 장르(밴드
   애니송) 카탈로그에서 공식 BPM의 59%가 ×2 쪽에 있다는 사전분포가 지배적이기 때문.
4. 잔여 한계: base 주파수 자체가 틀린 3.1%는 어떤 옥타브 규칙로도 못 구제 — 이건
   메인 로컬에서 스템 재분석(또는 bestdori 값 직접 채용)이 필요하다.

**판정: 유의미하게 낮은 오차 — 메인 로컬 전곡 적용 가치 있음.** 단, 적용 시 τ 규칙이
아니라 최대 후보 규칙을 쓰고, bestdori 매칭 가능곡(573곡)은 공식 BPM을 그대로 쓰는 것이
가장 정확하다(추정은 미매칭 88곡에만 필요).
