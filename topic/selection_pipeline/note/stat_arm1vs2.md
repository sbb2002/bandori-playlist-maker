# 참고용 인수인계 — arm1 vs arm2 통계검정 결과 (2026-07-18 별도 세션 분석)

> 이 문서는 `bandori-playlist-maker` 저장소 `research` 브랜치의
> `topic/selection_pipeline/report/01-selection_pipeline_comparison.md` §0~§6에 대한
> **후속 통계검정**이다. 원 보고서는 arm1(4.333) vs arm2(4.125) 평균값을 단순 대소비교만 했는데,
> 별도 세션에서 원본 데이터(`out/method3_blind_sheet.csv` + `out/method3_blind_mapping.csv`)를
> 직접 파싱해 정식 유의성 검정과 검정력(power) 분석을 수행했다. 본 보고서를 수정하는 게 아니라
> **참고 자료로 추가**하는 용도다.

## 데이터·방법

- `method3_blind_sheet.csv`(eval_id별 1~5 Likert 점수) + `method3_blind_mapping.csv`(eval_id →
  query_id, 소속 arm)를 eval_id 기준으로 조인.
- `arms` 컬럼이 `arm1|arm2`처럼 파이프 구분으로 여러 arm에 동시 소속된 행이 있어(같은 곡이 두
  arm에서 동시에 뽑힌 경우), 각 소속 arm마다 행을 복제해 long-format으로 펼침 → arm1 n=24,
  arm2 n=24, arm3 n=24 (원 보고서 표와 일치).

## 검정 결과

**1) 항목 단위 독립표본 비교 (arm1 24개 vs arm2 24개)**
- Mann-Whitney U = 317.0, **p = 0.522**
- Welch's t-test: t = 0.866, **p = 0.392**
- 효과크기: rank-biserial r = −0.101, Cohen's d = 0.250 (작음)

**2) 쿼리 단위 대응비교 (쿼리별 평균 점수, n=8쌍)**
- Wilcoxon signed-rank: W = 7.0, **p = 0.266**
- Paired t-test: t = 0.886, **p = 0.405**
- 방향 불일치: 8쌍 중 6개는 arm1 우위지만 Q5는 arm2가 크게 우세(−1.333) — 효과가 방향성조차
  일관되지 않음.

→ **두 검정 모두 p > 0.05로 유의한 차이 없음.** "arm2 ≤ arm1이므로 구조 기각"이라는 원 보고서
결론 자체는 뒤집히지 않지만, "arm1이 arm2보다 통계적으로 우월하다"고 주장할 근거도 없다 — **두
arm은 통계적으로 동등한 성능**이라는 게 더 정확한 표현이다.

## 검정력(power) 분석 — 몇 문항이 있어야 유의성을 볼 수 있는가

α=0.05, power=80% 기준:

| 설계 | 관측 효과크기 | 80% 검정력에 필요한 n | 현재 n의 실제 검정력 |
|---|---|---|---|
| 독립표본(항목 단위, arm당) | Cohen's d = 0.250 | **arm당 253개 항목** (쿼리당 top-3 유지 시 **쿼리 85개**) | 13.6% |
| 대응표본(쿼리 단위) | dz = 0.313 | **쿼리(쌍) 82개** | 12.0% |

역으로, 현재 n(8쿼리/24항목)으로 80% 검정력을 확보하려면 d ≥ 0.826(또는 dz ≥ 1.156) 수준의
아주 큰 효과가 필요한데, 실측 효과(d=0.25, dz=0.31)는 이보다 훨씬 작다. 즉 **지금 표본으로는
"차이가 작다/없다"와 "차이가 있는데 못 잡았다"를 구별할 수 없는 근본적인 검정력 부족 상태**다.

## 원 보고서와의 연결점 — §6.4 재확인

원 보고서 §6.4는 "상황/기능성 쿼리(Q5,Q6)에서 arm2가 arm1보다 크게 우세(+0.667, 인트로 오인
제외 후)"를 발견했다. 이는 전체 표본을 다 늘리는 것보다 **이 서브그룹만 표적으로 재검증 쿼리를
추가하는 편이 훨씬 효율적**이라는 뜻이다. 전체 82~85개 규모 재검증보다, 상황/기능성 카테고리에
집중한 소규모 추가 블라인드 테스트가 우선순위가 높을 수 있다.
