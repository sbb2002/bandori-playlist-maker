# 선곡 파이프라인 재검증 v2 — arm1 vs arm2, 84쿼리 확장 결과

> 설계: `topic/selection_pipeline/DESIGN_v2.md`(사전 등록, §0 판정표). `report/01-*.md` §7이
> 지적한 검정력 부족(원래 n=8쿼리, 필요 n=82쿼리)을 해소하기 위한 확장 재검증. arm3는 이번
> 라운드에서 제외(v1에서 이미 구조 기각 확정). 블라인드 채점(1~5 Likert) 84쿼리 × top-1,
> 실제 고유 147쌍, 연구자 1인, 2026-07-18 완료.

## 0. 사전등록 판정 결과 — **판정 보류 (표본 재확대로도 결론 안 남)**

| arm | 전체 평균 | n(항목) |
|---|---|---|
| arm1(프로덕션 재현) | 3.321 | 84 |
| arm2(가사 후보추림+절대강도) | **3.571** | 84 |

DESIGN_v2.md §0 판정표:
- 조건 (a) "arm2 평균 > arm1 평균" → **충족** (3.571 > 3.321, +0.250)
- 조건 (b) "Wilcoxon signed-rank(쿼리 단위 대응비교, 양측) p < 0.05" → **불충족** (p = 0.109)

(a)만 충족하고 (b)가 불충족이므로 사전등록 판정표에 따라 **"판정 보류(표본 재확대 필요)"** —
구조 채택도, 구조 기각도 이 라운드 단독으로는 선언할 수 없다.

## 1. 검정 결과

**항목 단위(독립표본, arm1 84개 vs arm2 84개)**
- Mann-Whitney U = 3150.0, p = 0.2145
- Welch's t-test: t = −1.307, p = 0.1929
- Cohen's d(arm2−arm1) = 0.202(작음)

**쿼리 단위(대응비교, n=84쌍 — §0 판정 기준)**
- Wilcoxon signed-rank: W = 343.0, **p = 0.1094**
- Paired t-test: t = −1.652, p = 0.1024
- 평균 차이(arm2−arm1) = +0.250, dz = **0.180**
- 방향: arm2 우위 26쌍, arm1 우위 17쌍, **동점 41쌍**

## 2. 검정력 재계산 — 이번에도 부족했다 (원인: 관측 효과크기가 더 작아짐)

`report/01-*.md` §7.3은 v1의 관측 효과(dz=0.313)를 기준으로 84쿼리면 80% 검정력이 나올
것으로 계산했다. 하지만 v2에서 실제 관측된 효과크기는 **dz=0.180**으로 v1보다 작았다 —
그 결과:

| 지표 | 값 |
|---|---|
| 이번 라운드 dz | 0.180 |
| n=84에서 실제 검정력 | **37.2%** |
| 80% 검정력에 필요한 n | **약 244쿼리(쌍)** |

즉 "82~85쿼리면 충분하다"는 계획은 **v1의 효과크기가 재현된다는 가정 위에서만 성립**했는데,
그 가정이 틀렸다(효과가 더 작았다) — DESIGN_v2.md §1이 미리 경고했던 바로 그 시나리오다.
244쿼리 규모의 3차 라운드는 현실적으로 무리라고 판단한다.

## 3. 데이터 품질 문제 — 쿼리 대비 곡 다양성 붕괴 (사후 발견, 결과 해석에 영향)

채점 도중 연구자가 "쿼리는 다른데 곡이 계속 겹친다"고 지적해 확인한 결과, 실제로 심각한
쏠림이 있었다:

- arm1: 쿼리 84개인데 실제 서로 다른 곡은 **26개뿐**(`roselia__623`·`raise_a_suilen__491`
  각각 13회 반복 선곡).
- arm2: 84개 중 **35개** 곡(가사 후보추림이 있어 arm1보다는 덜하지만 여전히 심함).
- **147개 고유 (쿼리,곡) 쌍 중 21쌍은 arm1·arm2가 아예 같은 곡을 선곡**했다 — 이 21쌍은
  Wilcoxon에서 자동으로 동점 처리된다(§1의 동점 41쌍 중 상당수가 여기서 발생).

**원인**: arm1/arm2 알고리즘은 "쿼리의 intensity_target에 가장 가까운 곡"을 결정론적으로
고르는 방식인데, 밴드 미지정 쿼리 63개(카테고리 B·D)가 LLM 파싱을 거치며 **10개의 이산
intensity_target 값**(0.0~1.0, 0.1 단위 근처)으로 수렴했다 — 이는 파이프라인 버그가 아니라
**쿼리 설계 문제**다: 카테고리 B("밴드미지정+절대강도")·D("밝기재확인")에 문장 표현만 다르고
알고리즘이 보는 단일 스칼라(intensity_target) 기준으로는 사실상 동의어인 쿼리를 21개씩
채워넣은 것 — DESIGN_v2.md §2를 설계할 때 "표현 다양성"과 "신호 다양성"을 혼동한 실수.
top-K를 3→1로 축소한 것(§7 참조)이 이 붕괴를 곡 하나로 완전히 수렴시켜 노출을 극대화했다.

**해석에 미치는 영향**: 각 (쿼리,점수) 관측치 자체는 여전히 유효한 판단이지만(연구자가 매번
그 쿼리 텍스트 기준으로 채점), 같은 곡을 반복해서 들으며 채점했으므로 앵커링 편향(이전 인상이
다음 채점에 스며듦) 위험이 있고, 대응비교의 독립성 가정도 이상적이지 않다. §0의 "판정 보류"
결론에 이 캐비어트를 더하면 — **v2 결과는 액면 그대로도 유의하지 않지만, 곡 다양성 붕괴로
인한 편향 가능성까지 고려하면 신뢰도를 더 낮춰서 봐야 한다.**

## 4. v1과 종합 — 두 독립 라운드 모두 결론 없음 (오히려 이게 핵심 발견)

| | v1(8쿼리) | v2(84쿼리) |
|---|---|---|
| arm1 평균 | 4.333 | 3.321 |
| arm2 평균 | 4.125 | 3.571 |
| 명목상 우위 | **arm1** | **arm2** |
| 쿼리 단위 검정 p | 0.266(Wilcoxon) | 0.109(Wilcoxon) |
| 유의미? | 아니오 | 아니오 |

**두 라운드의 명목상 우위 방향이 정반대로 뒤집혔다** — v1은 arm1이 근소 우위, v2는 arm2가
근소 우위. 둘 다 통계적으로 유의하지 않다. 이 자체가 유의미한 관찰이다: **arm2가 arm1보다
일관되게 낫다는 근거는 두 번의 독립적 시도에서 모두 나오지 않았고, 방향성조차 재현되지
않았다** — 진짜 효과가 있다면 최소한 방향은 일관돼야 하는데 그렇지 않다는 건, 관측된 차이가
표본별 노이즈일 가능성이 높다는 뜻이다.

(참고: v1은 척도가 다른 표본이 아니라 같은 방법론의 초기 파일럿이었고, v2는 LLM 모델이 다르다
는 편차가 있다(`DESIGN_v2.md` §7) — 완전히 같은 조건의 반복은 아니다. 그럼에도 "두 번 다
유의하지 않고 방향도 다르다"는 사실 자체는 그대로 유효하다.)

## 5. 결론 및 권고

1. **arm2 프로덕션화는 이 시점에서 근거가 더 약해졌다.** v1 §6이 제기했던 "특정 카테고리에서
   arm2가 근소 우위"라는 후속 제안은, 더 큰 표본(v2)에서도 전체적으로 유의성을 얻지 못했고
   방향성도 재현되지 않았다 — `report/01-*.md` §6.5의 "재검증 없이 프로덕션화 금지" 권고가
   실제로 재검증까지 거쳤음에도 결론이 안 났다는 뜻이므로, **이 경로는 여기서 접는 것을
   권장한다.**
2. **`selection.py`는 현행 로직(arm1) 유지.** 이건 이미 v1 §0의 공식 결론이었고, v2도 이를
   뒤집을 근거를 제공하지 못했다(오히려 "arm2가 더 낫다"는 방향조차 재현 안 됨을 추가로 확인).
3. 244쿼리 규모의 3차 라운드는 권장하지 않는다 — 비용 대비 기대 이득이 낮고(관측 효과가
   작고 라운드마다 방향이 바뀜), §3의 쿼리 설계 결함을 먼저 고치지 않으면 3차 라운드도 같은
   문제(곡 다양성 붕괴)를 반복할 것이다.
4. 이 주제(선곡 파이프라인 가사 후보추림)는 **여기서 종결을 권고**한다 — v1 구조 기각(§0) +
   v2 방향 불일치·비유의(본 보고서)로 이미 두 번의 독립적 시도가 모두 "채택" 방향 증거를
   내지 못했다. 추가 라운드보다는 다른 개선 축(예: valence 축 부재 — `report/01-*.md` §2b,
   [[vector-embedding-phase2-status]])에 리소스를 옮기는 편이 낫다.
