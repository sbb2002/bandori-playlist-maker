# 배포 시 시나리오

1. 첫 공개 시 유저들이 몰려 큐가 동시다발적으로 몰리면 openrouter의 ratelimit을 초과할 가능성이 있다.

    - 대안 제시 : 백엔드에 큐가 들어오면 큐를 저장해두고 순차적으로 ratelimit을 어기지 않는 선에서 빠르게 처리.

    - 큐에 올라간 사용자들의 순번을 순서가 다가올 때마다 업데이트해서 알려주기.

    - 모든 큐 소화 시 약 5분간 깨어있도록 핑을 쏘기. 새로운 큐가 들어오게 되면 이는 초기화.

2. render가 무료플랜이여서 콜드슬립 시 첫 실행은 약 2분의 추가 대기시간이 발생할 수 있음.

    - 콜드슬립 중 들어온 사용자에게는 '잠에서 꺠고 있어요오....zzZ / 유메와 빠와ㅡ!!!' 등과 같은 위트있는 멘트로 대기안내를 해주기.

3. render의 월간 무료플랜 초과 시 git issues나 telegram을 통해서 개발자인 나에게 알려주기.
