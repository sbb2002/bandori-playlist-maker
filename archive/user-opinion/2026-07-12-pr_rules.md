# PULL REQUEST에 대한 고찰

- 현재 베타 서비스가 시작 중이여서 이제부터 바로 main merge 없이 PR을 거친 후 개발자인 내가 검수 후 PR 승인함.

- PR의 목적은 사용자들에 대해 merge, 버전 패치 등으로 인해 이 앱을 사용할 수 없는 상황을 최소화하기 위함임.

- 따라서 AM 04:00에 github actions를 이용해 자동으로 PR을 merge 처리하고자 함. 단, 내가 actions 동작 전에 PR을 반려처리하면 PR queue가 비게 되므로 그냥 종료.


# 본 제안 관련 추가 기능 제안

- 형제 프로젝트인 bandori-song-sorter의 actions를 이용한 auto-loader를 통해 신곡을 자동 추가하고 있음.

- 똑같은 원리를 이용해 신곡 추가 시 형제 프로젝트처럼 내가 미리 만든 파일을 실행해 신곡 다운로드 및 분석 후 PR 요청까지 수행.

- 그러면 AM 04:00에 actions에서 기존 PR과 데이터 PR을 동시에 머지함.

- 단, 형제 프로젝트의 신곡 분석방법은 현재 앱의 활용 변수가 다르기 때문에 분석방법을 현 프로젝트에 맞게 수정해야함.

- AM 04:00에 2개 이상의 PR이 발생할 경우 순서대로 수행. 핫픽스 등의 패치성 PR은 작업 당시 최신화 브랜치를 기준으로 작업하기 때문에 파일 충돌을 없앨 계획. 데이터 추가성 PR은 데이터만 추가되는 것으로 코드충돌 우려없음. 만약 패치성 PR이 2개 이상인 경우 PR을 시간 순으로 수행하는 것이 중요함.


# 핫픽스

- 1회 사용자 요청 후 2회 요청 시 1회에서 사용한 파라미터가 또 유지됨.

- F5를 눌러 새로고침 후 다시 요청했는데 마찬가지로 1회의 파라미터 세팅이 유지됨. 그 증거로 Summary card가 토씨하나 안틀리고 똑같음.

- '사용자 요청 프롬프트'와 '세부 설정'의 로직을 변경하고자 함.

    * race condition or passive control  : 사용자 요청 시 모델이 세부설정을 제어함. 이후 2회 요청 시 프롬프트를 바꾸면 해당 세부설정을 제어하지 않음. 이는 경합 문제거나 1회성 소극적 제어로 판단되고, 내 말이 맞다면 이는 우선순위를 명시하지 않아서 생긴 것이라 사료됨.

    * 위의 내용이 사실이라면 아래와 같이 동작하도록 수정하고자 함.
        1) 사용자 프롬프트 & 세부 설정 탭 닫힘 = 세부 설정을 사용하지 않는 것으로 간주하고, 모델이 세부설정 파라미터를 기존 로직대로 제어.
        2) 사용자 프롬프트 & 세부 설정 탭 열림 = 사용자가 세부 설정을 설정한 것으로 간주하여, 사용자가 설정한 파라미터를 우선으로 하되 모델이 다음 파라미터를 변동할 수 있음.
            * 에너지 텐션 그래프의 각 구간의 에너지 값을 기존값±0.05 내외로 수정 가능.
            * 나머지 파라미터는 사용자의 설정대로 고정할 것.
        3) 2회 이상 사용 시 사용자 프롬프트(이전 회차의 프롬프트 텍스트가 달라짐) & 세부 설정 탭 열림 = 모델에게 이전 회차 텍스트와 현재 회차 텍스트를 동시에 제공하여 본질적으로 같은 내용인 경우 `2)`와 동일하게 동작. 만약 다른 내용이라고 판단될 경우 `1)`와 동일하게 동작.

    * 세부 설정 탭을 열면 이 탭에서 '여기서 직접 플레이리스트 메이킹을 제어할 수 있어요~ 완전 자동으로 플리를 만들고 싶다면 이 탭을 닫아주세요!'라고 안내하는 문구를 추가해줘.


# 핫픽스 제안 2

- 세부설정을 아래와 같이 패치.

    * UI
        - 텍스트박스 '에너지 단계 수'를 지우고 에너지 텐션 그래프에 아래 기능을 추가하여 대체하고자 함.
            * 그래프의 빈 부분에서 마우스 우클릭(모바일은 1초간 홀딩) - [구간 추가] 선택 : 해당 x좌표에 에너지 구간 분리선을 하나 더 추가.
            * 그래프의 에너지 구간 포인트(마름모)를 마우스 우클릭(모바일은 1초간 홀딩) - [구간 제거] 선택 : 해당 분리선을 없애고, 에너지 레벨 값은 다음 구간의 에너지값으로 대표함.
            * 에너지 구간 분리선은 최대 10개까지 지원. 10개 초과 시 '구간 분리선은 최대 10개까지 만들 수 있어요!'라고 안내메시지 띄우기.
